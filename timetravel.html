History of Hyperlink

  • 1945 - Concept of linking information

    An Essay "As We May Think" was published. In this essay, Vannevar Bush described the concept of linking information together in a "trail" using a microfilm-based machine called the Memex.

  • 1964/1965 - The term "link" and theoretical worldwide computure network

    Ted Nelson, inspired by Bush's ideas, coins the term "link". From 1964 to 1980, he transposed Bush's concept of automated cross-referencing into the computer context, made it applicable to specific text strings rather than whole pages. Ted Nelson generalized the concept from a local desk-size machine to a theoretical proprietary worldwide computer network. He then advocated the creation of such network.

  • 1966 - First implementation

    A team led by Douglas Engelbart implemented the hyperlink for scrolling within a single document, and soon in 1968 for connecting paragraphs within speparate documents.

  • 1983 - Highlighted links in HyperTIES system

    Ben Shneiderman and graduate student Dan Ostroff implement highlighted links in the HyperTIES system. The HyperTIES system was used to created the world's first eletronic journal.

  • 1987 - Hyperlinking between various pages

    Apple releases HyperCard for the Macintosh, allowing hyperlinking between pages within a document and to other documents and applications.

  • 1990 - Widespread use of hyperlink and visually effect

    Windows Help, introduced with Microsoft Windows 3.0, features widespread use of hyperlinks within help files. Also visually different kind of hyperlink was used.

  • 1991 - First widely adopted open protocal

    The Gopher protocol becomes the first widely used open protocol to include hyperlinks between Internet sites.

  • 1993 - HTML and browser

    The Mosaic browser is released, which can handle both Gopher and HTML links. HTML's advantage of mixing graphics, text, and hyperlinks leads to its widespread adoption.

  • 2000s - Dynamic routing in single page

    Development of modern web frameworks like React and Vue enables efficient creation of single-page applications with dynamic routing and navigation, facilitating seamless linking between various views within the same page.
